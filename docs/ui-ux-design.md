# UI/UX 설계서 — SmartHome IoT 관제 시스템

- 프론트: **React + TypeScript**, Floor Map은 **Konva**, 실시간은 WebSocket
- 근거: [iot_smarthome_srs.md](../iot_smarthome_srs.md) §2·3.2·5, [PROJECT_RULES.md](../PROJECT_RULES.md), [api-spec.md](api-spec.md)
- 상태: 초안 v0.1 (2026-07-09)

---

## 1. 설계 원칙

- **공간 인지형(Spatial-aware)**: 도면 위에서 기기를 직관적으로 파악·제어(SRS 1.1).
- **역할 기반 화면**: RBAC(§SRS 2)에 따라 메뉴·액션을 노출/차단. 권한 없는 액션은 화면에서 숨김 + 서버 재검증.
- **실시간 우선**: 상태/알람/명령 진행은 WebSocket push, 폴링 지양.
- **안전한 제어**: 파괴적·고위험 제어는 확인 단계 + HITL 승인 흐름을 UI로 강제(§SRS 3.5).
- **일관된 상태 표현**: 색상·아이콘 규칙을 전 화면에서 동일 적용(§6).

## 4. 핵심 화면 상세

### 4.1 실시간 관제 (Floor Map 중심, SRS 5.1 · 3.2)

```
┌───────────────────────────────────────────────────────────────┐
│ SmartHome   [Site▾][Building▾][Floor▾]      🔔3  ●Online  user ▾│
├───────┬───────────────────────────────────────────┬───────────┤
│ Layers│                                           │ 요약       │
│ ☑ 조명 │        ┌──────────area: 거실──────────┐    │ 기기 128  │
│ ☑ 센서 │        │   ●light-01(녹)  ▲sensor(노) │    │ ●ON 74    │
│ ☑ HVAC│        │   ●tv(회)        ⬛valve(검) │    │ ●Warn 5   │
│ ☐ Grid│        └──────────────────────────────┘    │ ●Alarm 2  │
│       │   [+][-] Zoom   ✋ Pan                       │ ●Off 3    │
│ 상태  │                                           │ Net ●OK   │
│ 범례  │                                           │           │
└───────┴───────────────────────────────────────────┴───────────┘
```
- **Floor Map**: Zoom / Pan / Layer(토글) / Grid / Marker / Polygon(Area) 지원(SRS 3.2).
- **Marker 색상 = 기기 상태**(§6). Area는 Polygon으로 표시, 선택 시 하이라이트.
- 우측 **요약 패널**: 상태별 카운트·Offline·Network(SRS 5.1).
- Marker 클릭 → Popover(현재 상태·최근 이벤트/알람/제어·센서·설정, SRS 3.2) → "상세" → Drawer(§4.2).

### 4.1-mode. 도면 편집 모드 vs 실행(관제) 모드 — ADMIN

도면은 **두 모드로 명확히 분리**한다. 모드 전환은 **ADMIN에게만** 노출되며, 기본은 실행 모드다.
모드에 따라 마커의 클릭·드래그 의미가 완전히 달라져 **오조작을 원천 차단**한다.

```
┌ 도면  [ 실행 ● | 편집 ○ ]  (ADMIN)         ...변경 3건  [취소] [저장] ┐
├───────────────────────────────────────────────────────────────────┤
│  ☑Grid ☑Snap        ┌──── area: 거실 ────┐                         │
│  [＋기기 배치]        │  ⤢light-01  ⤢sensor │  ← 드래그로 위치 이동    │
│  [✎ Area 편집]       │  ⤢tv                │                         │
│                      └─────────────────────┘                         │
└───────────────────────────────────────────────────────────────────┘
```

| 구분 | 실행(관제) 모드 — 기본 | 편집 모드 — ADMIN |
|---|---|---|
| 목적 | 실시간 상태 감시·제어 | 도면/배치 구성 |
| Marker 클릭 | Popover/제어(권한 시) | 선택(속성 편집) |
| Marker **드래그** | **비활성**(실수 이동 방지) | **위치 변경(드래그앤드롭)** |
| 제어(ON/OFF 등) | 활성(권한 시) | **비활성**(편집 중 제어 차단) |
| Area(Polygon) | 표시만 | 생성/정점 편집/삭제 |
| 실시간 상태색 | 실시간 갱신 | 표시하되 편집 우선 |

**편집 모드 동작(드래그앤드롭 배치)**
- 마커를 도면 위에서 드래그 → 새 좌표. **Grid Snap** 켜면 격자에 스냅.
- 새 기기 배치: `＋기기 배치` → 미배치 기기 목록에서 도면으로 드롭 → 좌표·Area 자동 결정
  (드롭 지점이 속한 Polygon으로 `areaId` 매핑).
- 좌표는 도면 픽셀(pos_x, pos_y) ↔ 실제 스케일(scale_m_per_px) 변환(§5).
- **Dirty 상태 관리**: 변경 건수 배지, `저장`/`취소`. 저장 전 이탈 시 확인 다이얼로그.
- **저장**: 변경분을 한 번에 커밋 → `PATCH /floors/{id}/layout`(배치 저장, §api 3). 낙관적 반영 + 실패 롤백.
- 변경은 감사 대상(DEVICE_RELOCATE 등) → `audit_log`.
- 편집 모드는 **ADMIN 전용**. USER/MONITOR/HITL에게는 토글 자체가 보이지 않는다.

### 4.2 Device History Drawer (SRS 5.4)

```
┌── light-01 · 거실 조명 ─────────────── ✕ ┐
│ 상태 ●ON   [ OFF ] [ ON ]  (권한 CONTROL) │
├───────────────────────────────────────────┤
│ [최근24h이벤트][제어][알람][센서][조치]  ← 탭 │
│ 10:22 관리자 turn_off  SUCCEEDED           │
│ 10:05 AI 추천 소등 (승인대기)              │
│ 09:40 Warning: 과열 임계 근접              │
│ ─ 센서 미니차트 (온도/전력) ─              │
└───────────────────────────────────────────┘
```
- 탭: 최근24h 이벤트 / 최근 제어 / 최근 알람 / 센서 데이터 / 사용자 조치.
- 상단 ON/OFF는 **CONTROL 권한**일 때만 활성. 실행 시 명령 진행 상태(PENDING→…) 인라인 표시.

### 4.3 System Timeline (SRS 5.2)

```
10:21 │ 🤖 AI 추천 생성 (전체 소등, conf 0.72)
10:22 │ 👤 관리자 승인
10:22 │ ⚙ 거실 조명 OFF (CMD-...014)
10:23 │ ✅ 성공
```
- 실시간 이벤트를 시간순 스트림으로. 아이콘으로 actor(AI/USER/SYSTEM)·유형 구분.
- 항목 클릭 → 관련 명령/알람/추천 상세로 이동.

### 4.4 데이터 컨텍스트 차트 (SRS 5.3)

```
 값 ▲
    │        ╱╲       ◆AI추천
    │   ╱╲  ╱  ╲   ▲알람
    │ ╱   ╲╱    ╲╱ ●사람개입
    └───────────────────────▶ 시간
```
- 센서 시계열 그래프 위에 **Alarm / Human Intervention / AI Recommendation** 마커를 오버레이.
- 마커 hover → 해당 이벤트 요약, 클릭 → Timeline/History 연동.

### 4.5 알람 센터 (SRS 3.3 · 2.2/2.3)

```
┌ 알람 ─ [전체][Reactive][Proactive][Optimization] ─ [미확인만] ┐
│ ● CRITICAL  gas-valve-01  가스누출   RAISED   10:31  [📷현장][확인][조치]│
│ ▲ WARNING   sensor-12    배터리30%  ACK      09:12  [Snooze]         │
│ ○ INFO      light-3F     불필요조명  RAISED   08:40  [확인]           │
└───────────────────────────────────────────────────────────────────┘
```
- tier·severity 필터, 색상 코딩(§6.2). 액션: 확인(Ack)/Snooze(USER)/조치·해제(MONITOR).
- 에스컬레이션 진행 배지 표시.
- **📷현장**: 발생원을 커버하는 카메라가 있을 때만 노출 → 라이브 뷰(§4.5-cam) 오픈(옵션 기능).

### 4.5-cam. 라이브 카메라 뷰 + PTZ (옵션 기능)

알람 행/기기 마커의 **"현장 확인"** 액션으로 열린다. 카메라가 없는 현장은 이 액션이 숨겨진다.

```
┌ 라이브 · cam-01 (거실) ─────────────── ✕ ┐
│ ┌──────────────────────────────────────┐ │
│ │            [ WebRTC 영상 ]            │ │
│ │                                      │ │
│ └──────────────────────────────────────┘ │
│        ▲                                   │
│   ◀  ⦿  ▶   Zoom [ − ][ + ]   ● REC       │
│        ▼        (PTZ, CONTROL 권한 시)     │
│ 프리셋: [현관][창가][전체]                 │
│ 연동 알람: ●CRITICAL 가스누출 10:31        │
└────────────────────────────────────────────┘
```
- PTZ 방향 패드/줌은 **CONTROL 권한**일 때만 활성. 조작은 명령 진행 상태로 피드백(§api 4-cam).
- 알람에서 진입 시, 정책의 `linkedCamera`/`autoGotoPreset`이 있으면 **자동으로 해당 프리셋**으로 이동된 화면을 먼저 보여준다.
- 영상은 미디어 서비스(WebRTC/HLS)에서 받고, 서명 URL 만료 시 자동 재발급.

### 4.6 HITL 승인 (SRS 3.5 — 안전 핵심)

```
┌ 승인 요청 ───────────────────────────────── ┐
│ 🤖 현재 외출 상태가 감지되었습니다.           │
│    모든 조명을 소등하시겠습니까?              │
│                                              │
│ 대상: 그룹 g-lights (12대)                    │
│ 신뢰도: 0.72   위험도: 고(전체 조명)          │
│ 근거: 09:38 이후 무재실 + 도어 잠금           │
│                                              │
│         [ 거절 ]        [ 승인 ]              │
└──────────────────────────────────────────────┘
```
- 고위험 장치(메인 차단기·도어락·가스·전체 조명) 또는 신뢰도 미만이면 **승인 전 실행 불가**.
- 승인/거절 즉시 결과 피드백 + Timeline 반영. 결정은 학습 데이터로 저장(백엔드).

### 4.7 스케줄러 (SRS 3.4)

```
┌ 스케줄 편집 ──────────────────────────────── ┐
│ 이름 [야간 소등]  대상 (○기기 ●그룹) [g-lights]│
│ 방식 [Cron ▾]  식 [0 23 * * *]                │
│ Payload {command: turn_off}                    │
│ 상태 [●활성]                     [저장][실행이력]│
└──────────────────────────────────────────────┘
```
- 방식: One Time / Daily / Weekly / Monthly / Cron / Event. 방식별 입력 폼 전환.

### 4.8 관리 화면 (ADMIN)

- **공간/평면도**: 평면도 업로드 → 스케일 지정 → Polygon으로 Area 그리기(SRS 2.1.1).
- **기기/그룹/UNS**: 기기 등록·위치 지정(도면 클릭으로 x,y)·Area 매핑·그룹(N:M)·UNS 토픽 확인.
- **카메라(옵션)**: `category=CAMERA` 등록 시 스트림 주소·프로토콜(RTSP/WebRTC/HLS/ONVIF)·PTZ 지원·
  설치 방향/화각 입력, PTZ 프리셋 정의, 커버 Area 매핑. 알람 정책에 카메라·자동 프리셋 연동 설정.
- **사용자/권한**: 사용자 생성, 역할 지정, Area/Device/Group 권한 매트릭스.

## 5. 도면 인터랙션 (Konva) — 모드별

| 기능 | 실행(관제) 모드 | 편집 모드(ADMIN) |
|---|---|---|
| Zoom | 휠/버튼, 커서 기준 | 동일 |
| Pan | 스페이스+드래그 / ✋ | 동일 |
| Layer | 레이어 토글(조명/센서/HVAC/Camera/Grid) | 동일 |
| Grid / Snap | Grid 표시 | Grid + **Snap**(격자 스냅) |
| Marker 클릭 | 선택/Popover(제어) | 선택(속성 편집) |
| Marker **드래그** | 비활성 | **드래그앤드롭 위치 이동** |
| 기기 배치 | – | 미배치 목록 → 도면으로 드롭 |
| Polygon(Area) | 표시 | 정점 클릭으로 생성/수정/삭제 |
| Camera(옵션) | 클릭=라이브 뷰 | 위치·heading·FOV 편집 |

- 마커 위치는 도면 좌표(pos_x, pos_y) ↔ 실제 스케일(scale_m_per_px) 변환.
- 카메라는 📷 마커 + 방향(heading)·화각(FOV) 부채꼴로 표시(별도 레이어 토글), PTZ/온라인 상태 병기.
- 모드 개념·저장 흐름은 §4.1-mode 참조. 편집 모드에서는 실시간 제어를 차단해 오조작을 방지한다.

## 8. 반응형 / 모바일

- 데스크톱: 3분할(레이어·도면·요약). 태블릿: 요약 접이식. 모바일(SRS 7): 도면 단일 뷰 +
  하단 시트로 Drawer/알람. 제어·승인 등 핵심 액션은 모바일 우선 보장.

## 9. 접근성

- 색+아이콘 병행, 키보드 내비게이션, 명령/알람 상태의 스크린리더 라벨, 대비 기준 준수.

## 10. 미해결/후속 (부록 A.2 연계)

- 디자인 시스템/컴포넌트 라이브러리 선정(예: 사내 토큰 vs 오픈소스)
- 모바일 앱(Android/iOS) 네이티브 vs 반응형 웹(SRS 7)
- 3D/BIM·디지털 트윈 뷰 통합 화면(SRS 7)
- 다국어(i18n) 정책
